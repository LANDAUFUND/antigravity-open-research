# СЛОВАРЬ АГРЕГИРОВАННЫХ ДАННЫХ АНТИГРАВИТАЦИИ

АНТИГРАВИТАЦИЯ РАССМАТРИВАЕТСЯ КАК ИССЛЕДОВАТЕЛЬСКОЕ НАПРАВЛЕНИЕ, ТРЕБУЮЩЕЕ МАТЕМАТИЧЕСКОЙ ФОРМАЛИЗАЦИИ, ЭКСПЕРИМЕНТАЛЬНОЙ ПРОВЕРКИ, ИЗМЕРИМОСТИ И ВОСПРОИЗВОДИМОСТИ.

## 1. НАЗНАЧЕНИЕ РАЗДЕЛА

Словарь задает публичные поля агрегированных данных, которые можно включать в DOI/архив без раскрытия raw data, закрытых режимов стендов, персональных данных и патентоспособных деталей.

## 2. БАЗОВЫЕ ФОРМУЛЫ

```text
P0 = m · g
η = ΔP / P0
z = ΔP / uΔ
ur = u / |x|
```

## 3. РАСШИФРОВКА ПЕРЕМЕННЫХ

- `P0` - контрольный вес.
- `m` - масса образца.
- `g` - ускорение свободного падения.
- `η` - относительное изменение веса.
- `ΔP` - изменение веса относительно контроля.
- `uΔ` - стандартная неопределенность изменения веса.
- `z` - нормированное отклонение.
- `ur` - относительная неопределенность.
- `u` - стандартная неопределенность величины.
- `x` - измеряемая величина.

## 4. ЕДИНИЦЫ ИЗМЕРЕНИЯ

- `P0`, `ΔP`, `uΔ`: ньютон, `Н`.
- `m`: килограмм, `кг`.
- `g`: метр на секунду в квадрате, `м/с²`.
- `η`, `z`, `ur`: безразмерно; `η` дополнительно допускается в `ppm`.
- Дата и время: ISO 8601.
- Температура: `К` или `°C` с явной шкалой.
- Давление: `Па`.
- Магнитная индукция: `Тл`.
- Электрическое поле: `В/м`.

## 5. ПРИМЕРЫ РАСЧЕТОВ

Если `m = 0.25 кг`, `g = 9.80665 м/с²`, то:

```text
P0 = 0.25 · 9.80665 = 2.4516625 Н
```

Если `ΔP = 0.00002 Н`, `uΔ = 0.00004 Н`, то:

```text
z = 0.00002 / 0.00004 = 0.5
```

Это совместимо с нулевым результатом и должно быть отражено в поле `СТАТУС_СЕРИИ`.

## 6. ЧТО ИМЕННО ДОЛЖНА ПРОВЕРИТЬ ЛАБОРАТОРИЯ

- Все ли поля имеют единицы.
- Нет ли полей, раскрывающих закрытые режимы стендов.
- Не смешаны ли raw data и агрегаты.
- Есть ли поле неопределенности для каждого физического результата.
- Есть ли статус серии и причина статуса.
- Есть ли связь с GitHub commit, Bitrix24 file ID или DOI/архивом.

## 7. КАКИЕ ДАННЫЕ НУЖНО СОБРАТЬ

| Поле | Назначение | Единицы | Публичность |
| --- | --- | --- | --- |
| `ID_АГРЕГАТА` | уникальный ID агрегата | текст | публично |
| `ID_СЕРИИ` | ID серии без raw data | текст | публично |
| `ID_ЛАБОРАТОРИИ` | обезличенный ID лаборатории | текст | публично или внутренне |
| `ID_СТЕНДА` | обезличенный ID стенда | текст | публично или внутренне |
| `КАТЕГОРИЯ` | установленная физика / компенсация / гипотеза / спекуляция | текст | публично |
| `МАССА_КГ` | масса образца | кг | публично, если безопасно |
| `P0_Н` | контрольный вес | Н | публично |
| `DELTA_P_Н` | отклонение веса | Н | публично |
| `U_DELTA_P_Н` | неопределенность отклонения | Н | публично |
| `ETA_PPM` | относительное отклонение | ppm | публично |
| `Z_SCORE` | нормированное отклонение | безразмерно | публично |
| `N_ИЗМЕРЕНИЙ` | число измерений | count | публично |
| `СТАТУС_СЕРИИ` | нуль / артефакт / требует повтора / опубликовано | текст | публично |

## 8. КАКИЕ ВЫВОДЫ ДЕЛАТЬ НЕЛЬЗЯ

- Нельзя делать вывод о новом физическом механизме из одного агрегата.
- Нельзя убирать неопределенность из публичной таблицы.
- Нельзя публиковать ID стенда, если он раскрывает закрытый лабораторный режим.
- Нельзя заменять отрицательный статус расплывчатым словом "перспективно".

## 9. КАК ОТЛИЧАТЬ НАУЧНЫЙ РАСЧЕТ ОТ ПСЕВДОНАУЧНОГО ЗАЯВЛЕНИЯ

Научный словарь фиксирует поля до интерпретации результата. Псевдонаучный словарь меняется после результата, чтобы убрать неудобные поля, неопределенность или отрицательные серии.

## 10. КАКИЕ СЛЕДУЮЩИЕ ДОКУМЕНТЫ НУЖНО ПОДГОТОВИТЬ

- Таблицу соответствия внутренних raw-полей публичным агрегатам.
- Словарь исключенных полей.
- Машиночитаемую JSON-schema для публичных агрегатов.
- Правило обезличивания лабораторий и стендов.
- Контрольный пример заполненного агрегата.
