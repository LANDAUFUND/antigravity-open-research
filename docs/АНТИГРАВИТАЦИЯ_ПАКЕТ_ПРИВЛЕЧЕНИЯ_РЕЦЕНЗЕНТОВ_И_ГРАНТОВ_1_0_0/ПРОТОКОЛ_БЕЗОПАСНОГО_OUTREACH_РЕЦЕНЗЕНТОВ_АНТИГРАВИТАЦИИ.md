# ПРОТОКОЛ БЕЗОПАСНОГО OUTREACH РЕЦЕНЗЕНТОВ АНТИГРАВИТАЦИИ

СТАТУС: `OUTREACH OPERATING PROCEDURE`.

ДАТА: `2026-05-25`.

## 1. ЦЕЛЬ

Найти и привлечь рецензентов без раскрытия закрытого лабораторного контура, без завышенных научных claims и без подмены review маркетингом.

## 2. ПОСЛЕДОВАТЕЛЬНОСТЬ

1. Выбрать роль review.
2. Найти кандидата и записать только код кандидата в реестр.
3. Проверить, что первое письмо не содержит закрытых материалов.
4. Отправить письмо с публичным пакетом и просьбой о готовности рассмотреть review.
5. Получить предварительное согласие или отказ.
6. Передать форму независимости и конфликта интересов.
7. Определить уровень доступа: `PUBLIC_PACKAGE_ONLY`, `INTERNAL`, `PROTECTED_UNDER_NDA`.
8. Зафиксировать бюджет или безвозмездный формат.
9. Передать материалы по журналу.
10. Получить письменное заключение.
11. Обновить матрицу `NC-001..NC-006`.

## 3. ЗАПРЕТЫ

Запрещено:

- писать, что антигравитация доказана;
- обещать революционный результат как факт;
- отправлять private repo до решения по доступу;
- отправлять Bitrix24 private links без необходимости;
- отправлять raw data до redaction;
- отправлять патентно значимые параметры до patent review;
- отправлять любые секреты доступа;
- давить на рецензента ради `GO`.

## 4. РАЗРЕШЕННЫЙ ПЕРВЫЙ ПАКЕТ

Первое касание может включать:

- краткое письмо;
- публичный GitHub;
- README направления;
- public release-candidate index;
- форму заключения;
- форму независимости;
- public/private boundary;
- вопрос о доступности и ставке.

## 5. СТАТУСЫ OUTREACH

Использовать только эти статусы:

- `NOT_CONTACTED`;
- `CONTACT_DRAFTED`;
- `CONTACTED`;
- `INTERESTED`;
- `DECLINED`;
- `CONFLICT_DISCLOSED`;
- `NDA_REQUIRED`;
- `NDA_SENT`;
- `NDA_SIGNED`;
- `MATERIALS_SENT`;
- `REVIEW_IN_PROGRESS`;
- `CONCLUSION_RECEIVED`;
- `CLOSED`.

## 6. ФИКСАЦИЯ РЕЗУЛЬТАТОВ

Каждое заключение должно иметь:

- роль;
- код рецензента;
- дату;
- список проверенных материалов;
- границы доступа;
- найденные замечания;
- блокирующие замечания;
- ограничения для public release;
- решение;
- подпись или подтвержденный идентификатор.

## 7. ПРАВИЛО ПО УМОЛЧАНИЮ

Если кандидат не ответил, роль не считается закрытой.

Если рецензент согласился, но не дал письменного заключения, роль не считается закрытой.

Если хотя бы один блокирующий review отсутствует, `RELEASE_1_0_0` остается `NO-GO ПО УМОЛЧАНИЮ`.
