## 1. ТЕМА

```text
Landau Fund: проверка release-candidate 1.0.0 по данным, лицензиям, патентам или security/privacy
```

## 2. УНИВЕРСАЛЬНОЕ ПИСЬМО

```text
Здравствуйте, [ИМЯ].

Landau Fund готовит открытый исследовательский пакет по направлению "Антигравитация". Пакет не утверждает доказанность антигравитации; он оформляет методологию, шаблоны данных, правила проверки, репликации, отрицательных результатов и release-candidate 1.0.0.

Нам нужен независимый review по роли [DATA / LICENSE / PATENT / SECURITY-PRIVACY / GRANT]. Задача - найти блокеры, ограничения, ошибки оформления и условия, при которых пакет можно или нельзя выпускать публично.

Мы работаем с разделением контуров:

- public: методологии, шаблоны, агрегированные данные, release notes, лицензии;
- internal: рабочие таблицы, review-решения, закрытая координация;
- protected: патентно значимые материалы, raw/private data, доступы;
- secret-critical: секреты инфраструктуры и учетные данные, которые не передаются рецензентам.

Если вы готовы рассмотреть такую проверку, мы пришлем индекс материалов, форму независимости, формат заключения и ожидаемый объем работы.

Допустимые решения:

- APPROVED;
- APPROVED WITH LIMITATIONS;
- REQUIRES FIXES;
- HOLD;
- NO-GO.

Решение NO-GO считается нормальным результатом review, если оно обосновано.

С уважением,
Команда Landau Fund
```

## 3. ВСТАВКА ДЛЯ DATA REVIEW

```text
В data review просим проверить manifest, размеры файлов, SHA-256, CSV/JSON-схемы, единицы измерения, статусы данных, воспроизводимость проверки и отсутствие raw/private data в публичном пакете.
```

## 4. ВСТАВКА ДЛЯ LICENSE REVIEW

```text
В license review просим проверить CC BY 4.0 для public non-code материалов, attribution, document notices, границу code/non-code и отсутствие автоматического публичного лицензирования закрытой лаборатории.
```

## 5. ВСТАВКА ДЛЯ PATENT REVIEW

```text
В patent review просим проверить, не раскрывает ли публичный пакет технические режимы, материалы, параметры, схемы или алгоритмы, которые требуют patent-first подхода или redaction.
```

## 6. ВСТАВКА ДЛЯ SECURITY / PRIVACY REVIEW

```text
В security/privacy review просим проверить секреты, персональные данные, внутренние ссылки, операционно чувствительные сведения, public/private boundary и необходимость redaction/removal.
```

## 7. ВСТАВКА ДЛЯ GRANT REVIEW

```text
В grant review просим оценить, насколько направление готово к грантовой заявке: цель, deliverables, бюджет, риски, отчетность, открытые результаты, независимая репликация и ограничения научных claims.
```
