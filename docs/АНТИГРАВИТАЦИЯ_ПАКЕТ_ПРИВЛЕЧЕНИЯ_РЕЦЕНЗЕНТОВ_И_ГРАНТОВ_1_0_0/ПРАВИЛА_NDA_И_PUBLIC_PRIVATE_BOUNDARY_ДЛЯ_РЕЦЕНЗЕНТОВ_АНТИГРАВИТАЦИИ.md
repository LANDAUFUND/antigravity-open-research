# ПРАВИЛА NDA И PUBLIC PRIVATE BOUNDARY ДЛЯ РЕЦЕНЗЕНТОВ АНТИГРАВИТАЦИИ

СТАТУС: `ACCESS BOUNDARY / NDA RULES`.

ДАТА: `2026-05-25`.

## 1. ПРИНЦИП

Рецензент получает только тот объем материалов, который нужен для его роли. Доступ расширяется поэтапно: сначала публичный пакет, затем внутренние материалы, затем protected-слой только при наличии основания, NDA и решения ответственного.

## 2. PUBLIC БЕЗ NDA

Можно отправлять без NDA:

- публичный GitHub `LANDAUFUND/antigravity-open-research`;
- README и публичный индекс;
- методологические документы;
- шаблоны лабораторных отчетов;
- шаблоны CSV без raw data;
- release-candidate и review-gate документы, если они уже опубликованы;
- форму независимости;
- форму заключения;
- public/private boundary note.

## 3. INTERNAL ТОЛЬКО ПО РОЛИ

Можно отправлять после назначения роли:

- рабочий список замечаний;
- review decision registry;
- статусы `NC-001..NC-006`;
- внутренние протоколы комиссии без секретов;
- агрегированные данные с удаленными персональными и патентными деталями.

## 4. PROTECTED ТОЛЬКО ПО NDA

Требует NDA и отдельного решения:

- raw data;
- закрытые лабораторные параметры;
- черновики патентных раскрытий;
- technical modes, materials, settings, diagrams, algorithms;
- неочищенные журналы экспериментов;
- private repo access;
- Bitrix24 private folder links;
- любые сведения, которые могут повлиять на patent-first стратегию.

## 5. SECRET-CRITICAL НЕ ПЕРЕДАЕТСЯ

Не передается рецензентам:

- пароли;
- API keys;
- GitHub tokens;
- 1Password items;
- cookies;
- SSH keys;
- recovery phrases;
- private infrastructure details;
- учетные записи сотрудников;
- административные Bitrix24 или GitHub доступы.

## 6. ОБЯЗАТЕЛЬНЫЕ ФОРМУЛИРОВКИ В NDA

NDA должен покрывать:

- запрет раскрытия protected materials;
- запрет самостоятельной публикации закрытых данных;
- запрет подачи патента на основе полученных материалов без письменного соглашения;
- обязанность сообщать о конфликте интересов;
- обязанность уничтожить или вернуть закрытые материалы по запросу;
- право рецензента дать отрицательное заключение;
- право проекта публиковать только согласованную публичную часть заключения.

## 7. ЧТО NDA НЕ ДОЛЖЕН ДЕЛАТЬ

NDA не должен:

- запрещать добросовестную научную критику публичных материалов;
- требовать положительного заключения;
- скрывать методологические ошибки;
- превращать гипотезы в доказанные утверждения;
- мешать публикации отрицательного результата, если он согласован как public output.

## 8. REDACTION ПЕРЕД ПЕРЕДАЧЕЙ

Перед передачей любого internal/protected файла:

1. удалить секреты доступа;
2. удалить персональные данные, если они не нужны роли;
3. скрыть патентно значимые параметры до решения patent reviewer;
4. заменить raw identifiers на коды;
5. сохранить SHA-256 переданной версии;
6. записать передачу в журнал.
