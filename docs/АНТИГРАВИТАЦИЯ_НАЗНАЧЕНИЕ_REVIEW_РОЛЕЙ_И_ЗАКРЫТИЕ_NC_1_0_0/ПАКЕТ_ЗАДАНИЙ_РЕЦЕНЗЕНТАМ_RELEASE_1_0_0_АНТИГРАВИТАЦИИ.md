# ПАКЕТ ЗАДАНИЙ РЕЦЕНЗЕНТАМ RELEASE 1.0.0 АНТИГРАВИТАЦИИ

СТАТУС: `ШАБЛОН ВЫДАЧИ ЗАДАНИЙ РЕЦЕНЗЕНТАМ`.

ДАТА: `2026-05-25`.

ПРОЕКТ: `LANDAU FUND`.

НАПРАВЛЕНИЕ: `АНТИГРАВИТАЦИЯ`.

## 1. ОБЩЕЕ ЗАДАНИЕ ДЛЯ ВСЕХ РЕЦЕНЗЕНТОВ

Проверить не "веру в направление", а конкретный документальный пакет `release-candidate 1.0.0`.

Рецензент должен подтвердить, ограничить или заблокировать выпуск. Рецензент имеет право написать `NO-GO`.

Рецензент не должен подтверждать доказанность антигравитации. Даже решение `GO` означает только допустимость выпуска конкретного публичного пакета с указанными ограничениями.

## 2. НАУЧНЫЙ РЕЦЕНЗЕНТ

Проверить:

- корректность физических формулировок;
- разделение установленной физики, инженерной компенсации веса, гипотез и спекулятивных моделей;
- наличие фальсифицируемости;
- отсутствие заявлений о доказанной антигравитации;
- корректность обращения с UAP-наблюдениями;
- запрет на подмену электромагнитных, тепловых, аэродинамических и вибрационных эффектов гравитационным выводом.

Выход:

```text
РЕШЕНИЕ: ОДОБРЕНО / ОДОБРЕНО С ОГРАНИЧЕНИЯМИ / ТРЕБУЕТ ИСПРАВЛЕНИЙ / ОТКЛОНЕНО
КЛЮЧЕВЫЕ ЗАМЕЧАНИЯ:
ОБЯЗАТЕЛЬНЫЕ ОГРАНИЧЕНИЯ ДЛЯ RELEASE NOTES:
```

## 3. DATA REVIEWER

Проверить:

- manifest;
- размеры файлов;
- SHA-256;
- CSV-схемы;
- воспроизводимость проверки;
- отсутствие raw/private данных в публичном пакете;
- наличие единиц измерения, погрешностей и статусов данных.

Выход:

```text
РЕШЕНИЕ:
ПРОВЕРЕННЫЕ ФАЙЛЫ:
НЕСОВПАДЕНИЯ:
ОГРАНИЧЕНИЯ ВОСПРОИЗВОДИМОСТИ:
```

## 4. LICENSE REVIEWER

Проверить:

- применимость CC BY 4.0 к public non-code материалам;
- отдельный статус code utilities;
- наличие attribution;
- отсутствие автоматического лицензирования private lab;
- document-level notices.

Выход:

```text
РЕШЕНИЕ:
ЛИЦЕНЗИОННЫЕ БЛОКЕРЫ:
ТРЕБУЕМЫЕ ИСПРАВЛЕНИЯ:
```

## 5. PATENT REVIEWER

Проверить:

- не раскрыты ли protected parameters;
- нет ли конкретных технических режимов, чертежей, материалов или алгоритмов, которые требуют patent first;
- отделены ли общие методологии от изобретательских раскрытий;
- не потерян ли приоритет через преждевременную публикацию.

Выход:

```text
РЕШЕНИЕ: PUBLISH / PUBLISH WITH REDACTIONS / PATENT FIRST / HOLD
ПАТЕНТНЫЕ БЛОКЕРЫ:
REDACTION LIST:
```

## 6. SECURITY / PRIVACY REVIEWER

Проверить:

- секреты;
- токены;
- пароли;
- private keys;
- cookies;
- персональные данные;
- приватную координацию;
- operationally sensitive details;
- соответствие public/private boundary.

Выход:

```text
РЕШЕНИЕ:
SCAN METHOD:
НАЙДЕННЫЕ РИСКИ:
REDACTION / REMOVAL ACTIONS:
```

## 7. RELEASE MANAGER

Проверить:

- закрыты ли `NC-001..NC-005`;
- есть ли письменные заключения;
- внесены ли ограничения в release notes;
- совпадают ли версии и хэши;
- не содержит ли финальный текст claim о доказанной антигравитации.

Возможные решения:

- `GO`
- `GO С ОГРАНИЧЕНИЯМИ`
- `NO-GO`
- `DEFER`

## 8. АРХИВАРИУС

Проверить:

- соответствие локального пакета;
- соответствие GitHub;
- соответствие Bitrix24;
- SHA-256;
- manifest;
- готовность DOI/архивного пакета.

Выход:

```text
ARCHIVE STATUS:
HASH STATUS:
VERSION STATUS:
DOI READY: YES / NO / AFTER FIXES
```
