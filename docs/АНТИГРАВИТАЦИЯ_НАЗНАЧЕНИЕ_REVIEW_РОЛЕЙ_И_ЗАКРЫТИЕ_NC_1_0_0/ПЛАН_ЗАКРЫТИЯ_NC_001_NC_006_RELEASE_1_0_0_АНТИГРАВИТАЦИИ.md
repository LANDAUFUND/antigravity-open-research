# ПЛАН ЗАКРЫТИЯ NC-001..NC-006 RELEASE 1.0.0 АНТИГРАВИТАЦИИ

СТАТУС: `ПЛАН КОРРЕКТИРУЮЩИХ ДЕЙСТВИЙ`.

ДАТА: `2026-05-25`.

ПРОЕКТ: `LANDAU FUND`.

НАПРАВЛЕНИЕ: `АНТИГРАВИТАЦИЯ`.

## 1. НАЗНАЧЕНИЕ

План переводит открытые несоответствия `NC-001..NC-006` из общего статуса `ОТКРЫТО` в конкретные действия, доказательства и критерии закрытия.

Пока все `NC-001..NC-006` не закрыты, финальный `RELEASE_1_0_0` не объявляется.

## 2. ПРИНЦИП ЗАКРЫТИЯ

Каждое `NC` закрывается только при наличии:

1. назначенного ответственного;
2. письменного заключения или протокола;
3. ссылки на проверенные артефакты;
4. решения по статусу;
5. даты закрытия;
6. записи в таблице закрытия;
7. отсутствия блокирующих замечаний.

## 3. NC-001: НАУЧНАЯ ПРОВЕРКА

Проблема: научный рецензент не назначен.

Корректирующее действие:

- назначить научного рецензента;
- выдать ему расчетный каркас, review-пакет, validation-пакет, replication-пакет и список запретных формулировок;
- получить письменное заключение.

Критерии закрытия:

- проверено, что документы не утверждают доказанность антигравитации;
- установленные эффекты отделены от гипотез;
- инженерная компенсация веса не названа гравитационным управлением;
- фальсифицируемость и ограничения явно описаны;
- рецензент имеет право поставить `NO-GO`.

## 4. NC-002: ДАННЫЕ И ВОСПРОИЗВОДИМОСТЬ

Проблема: ответственный за данные не назначен.

Корректирующее действие:

- назначить data reviewer;
- проверить manifest, размеры, SHA-256, CSV-схемы, release-candidate package и machine-evidence package;
- подтвердить, что public-пакет не содержит raw/private данных.

Критерии закрытия:

- manifest воспроизводим;
- SHA-256 совпадают;
- временные поля `ТРЕБУЕТ_*` отсутствуют в publishable manifest;
- raw private data не попали в public;
- ограничения данных зафиксированы.

## 5. NC-003: ЛИЦЕНЗИИ

Проблема: лицензии и граница public/private не закрыты review.

Корректирующее действие:

- назначить license reviewer;
- проверить `LICENSE`, `LICENSE.md`, document notices, MIT/code utility notices, атрибуцию и public/private boundary;
- подтвердить, что private lab не получает public license автоматически.

Критерии закрытия:

- public non-code материалы корректно идут под CC BY 4.0;
- code utilities имеют отдельный software license;
- private/internal материалы не лицензированы наружу по ошибке;
- attribution к Landau Fund указан.

## 6. NC-004: ПАТЕНТЫ

Проблема: патентная проверка не завершена.

Корректирующее действие:

- назначить patent reviewer;
- проверить public-пакеты на преждевременное раскрытие конкретных технических режимов, чертежей, материалов, алгоритмов и параметров;
- отметить, какие материалы можно публиковать, а какие требуют hold / patent first.

Критерии закрытия:

- научная гипотеза отделена от патентоспособного технического решения;
- protected parameters не раскрыты;
- patent-sensitive материалы остаются в private/protected контуре;
- решение `publish / patent first / hold` записано.

## 7. NC-005: БЕЗОПАСНОСТЬ И ПРИВАТНОСТЬ

Проблема: security/privacy review не завершен.

Корректирующее действие:

- назначить security reviewer;
- выполнить secret scan, privacy scan, operational sensitivity scan;
- проверить GitHub, Bitrix24 и локальные public/private границы.

Критерии закрытия:

- нет секретов, токенов, паролей, cookies, private keys;
- нет персональных данных без основания;
- нет приватной координации в public;
- нет sensitive lab parameters в public;
- результаты scan записаны.

## 8. NC-006: СВОДНОЕ РЕШЕНИЕ

Проблема: финальное `GO / NO-GO` решение не принято.

Корректирующее действие:

- назначить release manager;
- дождаться закрытия `NC-001..NC-005`;
- оформить сводный протокол.

Критерии закрытия:

- все блокирующие `NC` закрыты или явно переведены в `DEFER` с запретом финального релиза;
- все ограничения перенесены в release notes;
- решение не содержит утверждения доказанности антигравитации;
- протокол подписан ответственным release manager.

## 9. НЕДОПУСТИМЫЕ ОБХОДЫ

Нельзя закрывать `NC` через:

- устное согласие;
- отсутствие возражений;
- факт публикации в GitHub;
- факт загрузки в Bitrix24;
- наличие SHA-256 без человеческого review;
- общую фразу `научно корректно`;
- обобщение `секретов нет`, если scan не выполнен;
- давление сроком или публичной амбицией.

## 10. ТЕКУЩИЙ СТАТУС

На дату создания этого плана все `NC-001..NC-006` остаются открытыми до фактического назначения ролей и получения письменных заключений.

```text
RELEASE_1_0_0: HOLD / NO-GO ПО УМОЛЧАНИЮ
```
