# ЗАМЕТКИ РЕЛИЗА 1.0.0 АНТИГРАВИТАЦИИ

Статус: `RELEASE_CANDIDATE_DRAFT`.

Дата подготовки: `2026-05-25`.

Версия: `1.0.0-candidate`.

## 1. Назначение релиза

Релиз `1.0.0` должен стать первым стабильным публичным архивным пакетом направления `Антигравитация` в контуре Landau Fund. Его задача - зафиксировать методический и расчетный каркас, лабораторные шаблоны, протоколы контроля, отрицательные результаты как допустимый исход, межлабораторный обмен, метаанализ и DOI-готовую структуру архива.

Релиз не является заявлением о доказанной антигравитации.

Каноническая формулировка:

`АНТИГРАВИТАЦИЯ РАССМАТРИВАЕТСЯ КАК ИССЛЕДОВАТЕЛЬСКОЕ НАПРАВЛЕНИЕ, ТРЕБУЮЩЕЕ МАТЕМАТИЧЕСКОЙ ФОРМАЛИЗАЦИИ, ЭКСПЕРИМЕНТАЛЬНОЙ ПРОВЕРКИ, ИЗМЕРИМОСТИ И ВОСПРОИЗВОДИМОСТИ`.

## 2. Состав релиза

В релиз-кандидат входят следующие группы материалов:

- расчетный каркас и тематические формулы;
- паспорт лабораторного стенда;
- таблица измеряемых величин;
- протоколы калибровки;
- реестр стендов;
- контроль псевдонауки;
- независимая репликация;
- отрицательные и нулевые серии;
- межлабораторный обмен;
- метаанализ серий;
- DOI-архивный пакет;
- схема публичных агрегатов;
- скрипт проверки манифеста и хэшей;
- таблица соответствия Bitrix24 и GitHub.

## 3. Что изменилось к 1.0.0-candidate

- Создана связанная структура public/private контуров.
- Зафиксирована граница: публичные методики и агрегаты отделены от raw internal data.
- Добавлены Bitrix24 file IDs для архивного слоя.
- Добавлен контроль, что GitHub commit, Bitrix24 file ID, размер и SHA-256 не расходятся.
- Добавлен запрет трактовать DOI, GitHub или Bitrix24 как доказательство физического эффекта.

## 4. Проверка перед выпуском

Перед выпуском `1.0.0` нужно подтвердить:

- все release-candidate файлы имеют SHA-256;
- все release-candidate файлы имеют размер в байтах;
- все public файлы имеют путь в GitHub;
- все Bitrix24 file IDs проверены в группе `LANDAU.FUND | Лев Давидович Ландау`, group ID `620`;
- секреты, персональные данные и закрытые параметры стендов отсутствуют;
- лицензии разделены: non-code public materials - `CC BY 4.0`, code utilities - отдельная software license;
- патентоспособные технические решения не раскрыты без patent review;
- научные утверждения не выходят за пределы данных.

## 5. Данные, которых пока не хватает

- Реальный DOI или архивный URL после депонирования.
- Финальная дата review.
- Финальные имена reviewers.
- Полный public aggregate dataset после первых валидных серий.
- Подтвержденная таблица соответствия всех файлов релиза `1.0.0`, а не только DOI-архивного слоя.

## 6. Запрещенные формулировки

В release notes, README, metadata и публикациях нельзя использовать:

- `антигравитация доказана`;
- `получена антигравитационная тяга`;
- `UAP подтверждают технологию`;
- `секретность доказывает результат`;
- `DOI подтверждает физический эффект`.

Допустимая формулировка: `создан воспроизводимый исследовательский каркас и архивный пакет для проверки гипотез`.

## 7. Следующий шаг после release candidate

Следующий шаг - подготовить финальный `RELEASE_1_0_0` только после прохождения пяти проверок:

- scientific review;
- data review;
- license review;
- patent review;
- security review.
