# ГРАНИЦЫ МАШИННОЙ СВЕРКИ И ЧЕЛОВЕЧЕСКОГО REVIEW АНТИГРАВИТАЦИИ

СТАТУС: `ОГРАНИЧЕНИЯ ИНТЕРПРЕТАЦИИ`.

ДАТА: `2026-05-25`.

## 1. НАЗНАЧЕНИЕ

Документ отделяет машинную проверку файлов от научного, юридического, патентного и safety/security review.

Антигравитация рассматривается как исследовательское направление, требующее математической формализации, экспериментальной проверки, измеримости и воспроизводимости.

## 2. ЧТО МОЖЕТ ПРОВЕРИТЬ МАШИНА

Машинная сверка может проверить:

- наличие файлов;
- размеры файлов;
- совпадение SHA-256;
- наличие GitHub commit;
- наличие Bitrix24 file ID;
- соответствие локального файла скачанной Bitrix24-копии;
- базовую читаемость CSV;
- отсутствие известных паттернов секретов по заданному шаблону;
- отсутствие незакоммиченных изменений в целевых GitHub-репозиториях.

## 3. ЧТО МАШИНА НЕ МОЖЕТ ПОДТВЕРДИТЬ

Машинная сверка не может подтвердить:

- что антигравитация доказана;
- что гипотеза физически верна;
- что расчеты достаточны для эксперимента;
- что эксперимент воспроизводим;
- что артефакты измерения исключены;
- что патентный риск равен нулю;
- что лицензии полностью достаточны в каждой юрисдикции;
- что отсутствуют все возможные privacy-риски;
- что финальный релиз можно выпускать.

## 4. РАЗДЕЛЕНИЕ РЕШЕНИЙ

| Уровень | Кто закрывает | Что означает |
|---|---|---|
| Машинная сверка | Codex / автоматизированный скрипт | Файлы совпадают, IDs и хэши связаны |
| Научное review | Научный рецензент | Формулировки, модели, ограничения и фальсифицируемость проверены |
| Data review | Ответственный за данные | Данные, манифест, воспроизводимость и отсутствие подмен проверены |
| License review | Лицензионный проверяющий | Публичная лицензия и attribution проверены |
| Patent review | Патентный проверяющий | Патентные риски раскрытия оценены |
| Security/privacy review | Проверяющий безопасности | Секреты, приватные данные и границы public/private проверены |
| Release decision | Release manager | Принято `GO / GO С ОГРАНИЧЕНИЯМИ / NO-GO / DEFER` |

## 5. ЗАПРЕЩЕННАЯ ИНТЕРПРЕТАЦИЯ

Нельзя писать:

- `SHA совпал, значит научный результат доказан`;
- `файл есть в GitHub, значит релиз одобрен`;
- `файл есть в Bitrix24, значит review закрыт`;
- `секретов по паттернам не найдено, значит security review не нужен`;
- `release-candidate прошел strict manifest check, значит final release можно объявлять`;
- `инженерная компенсация веса равна антигравитации`.

## 6. РАЗРЕШЕННАЯ ИНТЕРПРЕТАЦИЯ

Можно писать:

```text
Машинная сверка файлов, хэшей, путей GitHub и Bitrix24 IDs выполнена.
Она подтверждает целостность и трассируемость пакета, но не заменяет scientific/data/license/patent/security review.
Текущий release decision остается NO-GO по умолчанию до закрытия review.
```

## 7. СЛЕДУЮЩЕЕ ДЕЙСТВИЕ

Передавать этот пакет рецензентам как входную доказательную базу, а не как финальное одобрение.
