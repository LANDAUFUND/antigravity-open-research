# ПРОТОКОЛ ПЕРЕДАЧИ ПАКЕТА РЕЦЕНЗЕНТАМ RELEASE 1.0.0 АНТИГРАВИТАЦИИ

СТАТУС: `ПОДГОТОВЛЕНО К ПЕРЕДАЧЕ, РЕЦЕНЗЕНТЫ НЕ НАЗНАЧЕНЫ`.

ДАТА: `2026-05-25`.

## 1. НАЗНАЧЕНИЕ

Документ описывает, как передавать машинно сверенный пакет рецензентам, не объявляя финальный релиз и не создавая видимость доказанности антигравитации.

Антигравитация рассматривается как исследовательское направление, требующее математической формализации, экспериментальной проверки, измеримости и воспроизводимости.

## 2. ЧТО ПЕРЕДАЕТСЯ

Рецензентам передаются:

- release-candidate package;
- review-gate package;
- operational-closure package;
- machine evidence registry;
- SHA-256 sums;
- список заданий по ролям;
- ограничения машинной сверки.

## 3. ОБЯЗАТЕЛЬНАЯ СОПРОВОДИТЕЛЬНАЯ ФОРМУЛИРОВКА

```text
Передаваемый пакет является машинно сверенной доказательной базой для review release 1.0.0.
Он подтверждает целостность файлов, соответствие GitHub/Bitrix24 контуров и SHA-256, но не является научным доказательством антигравитации и не закрывает review.
Текущее решение остается NO-GO по умолчанию до письменного закрытия scientific, data, license, patent и security/privacy review.
```

## 4. МИНИМАЛЬНЫЙ СОСТАВ ПЕРЕДАЧИ

| Роль | Что получает | Что возвращает |
|---|---|---|
| Научный рецензент | Расчетный каркас, review package, machine evidence | Scientific review заключение |
| Ответственный за данные | Release-candidate manifest, SHA-256, machine evidence | Data review заключение |
| Лицензионный проверяющий | Public repo, LICENSE, attribution, review package | License review заключение |
| Патентный проверяющий | Patent map, public/private boundary, review package | Patent review заключение |
| Проверяющий безопасности | Secret/privacy boundary, machine evidence | Security/privacy заключение |
| Release manager | Все заключения и журналы | `GO / GO С ОГРАНИЧЕНИЯМИ / NO-GO / DEFER` |

## 5. ЗАПРЕТЫ ПРИ ПЕРЕДАЧЕ

Нельзя:

- писать, что релиз уже одобрен;
- писать, что антигравитация доказана;
- подменять рецензию ссылкой на GitHub commit;
- подменять научное review совпадением SHA-256;
- передавать секреты, API keys, приватные контакты или operationally sensitive details;
- включать патентно чувствительные материалы без patent review.

## 6. КРИТЕРИЙ ПОЛУЧЕНИЯ ОБРАТНОГО РЕЗУЛЬТАТА

Пакет считается обработанным рецензентом только если получено письменное заключение с:

- именем роли;
- проверенной версией;
- ссылкой на входные артефакты;
- списком замечаний;
- статусом блока;
- датой;
- явным указанием ограничений.
