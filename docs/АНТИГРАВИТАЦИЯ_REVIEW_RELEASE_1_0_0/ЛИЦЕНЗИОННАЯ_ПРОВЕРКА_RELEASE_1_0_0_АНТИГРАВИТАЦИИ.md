АНТИГРАВИТАЦИЯ РАССМАТРИВАЕТСЯ КАК ИССЛЕДОВАТЕЛЬСКОЕ НАПРАВЛЕНИЕ, ТРЕБУЮЩЕЕ МАТЕМАТИЧЕСКОЙ ФОРМАЛИЗАЦИИ, ЭКСПЕРИМЕНТАЛЬНОЙ ПРОВЕРКИ, ИЗМЕРИМОСТИ И ВОСПРОИЗВОДИМОСТИ.

## 1. НАЗНАЧЕНИЕ

Документ задает license review перед финальным `RELEASE_1_0_0`. Цель - подтвердить, что публичные non-code материалы могут быть выпущены под `CC BY 4.0`, кодовые утилиты имеют отдельную software-лицензию, а закрытые лабораторные материалы не получают публичную лицензию автоматически.

## 2. ВХОДНЫЕ МАТЕРИАЛЫ

- `LICENSE`, `LICENSE.md`, `LICENSE-CODE-MIT`.
- `AUTHORSHIP.md`, `CITATION.cff`, `docs/LICENSE_POLICY.md`.
- Публичные docs, methodologies, templates, datasets-public.
- Private lab материалы как источник для проверки границы public/private, но не как материал автоматической публикации.

## 3. ОБЯЗАТЕЛЬНЫЕ ПРОВЕРКИ

- Проверить, что публичные текстовые материалы имеют статус `CC BY 4.0`.
- Проверить, что кодовые утилиты имеют явный `SPDX-License-Identifier: MIT` или иной явно выбранный software license.
- Проверить, что в публичный пакет не попали сторонние тексты, таблицы, изображения или данные без права публикации.
- Проверить, что private lab файлы не опубликованы автоматически и не смешаны с public release.
- Проверить, что есть обязательная атрибуция Landau Fund и ссылка на публичный GitHub-контур.
- Проверить, что адаптации и изменения можно отследить через changelog и commit history.

## 4. КРИТЕРИИ ПРОХОЖДЕНИЯ

License review проходит только если для каждого публичного файла понятен тип материала, источник, право публикации, лицензия, авторство и обязательная атрибуция.

## 5. СТОП-УСЛОВИЯ

- В public попал файл без понятного источника и права публикации.
- Private lab материал опубликован без release action.
- Кодовая утилита опубликована только под документной лицензией без software-лицензии.
- В документе отсутствует авторство, ссылка на репозиторий или license notice.
- Сторонний материал скопирован без разрешения или корректной лицензии.
