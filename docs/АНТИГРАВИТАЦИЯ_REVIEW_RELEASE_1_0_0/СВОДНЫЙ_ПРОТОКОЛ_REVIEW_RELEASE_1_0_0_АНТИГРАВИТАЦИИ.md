# СВОДНЫЙ ПРОТОКОЛ REVIEW RELEASE 1.0.0 НАПРАВЛЕНИЯ «АНТИГРАВИТАЦИЯ»

ДАТА: `2026-05-25`.

СТАТУС: `REVIEW GATE`, НЕ ФИНАЛЬНЫЙ `RELEASE_1_0_0`.

АНТИГРАВИТАЦИЯ РАССМАТРИВАЕТСЯ КАК ИССЛЕДОВАТЕЛЬСКОЕ НАПРАВЛЕНИЕ, ТРЕБУЮЩЕЕ МАТЕМАТИЧЕСКОЙ ФОРМАЛИЗАЦИИ, ЭКСПЕРИМЕНТАЛЬНОЙ ПРОВЕРКИ, ИЗМЕРИМОСТИ И ВОСПРОИЗВОДИМОСТИ.

## 1. НАЗНАЧЕНИЕ

Сводный протокол объединяет пять обязательных review-блоков перед возможным финальным `RELEASE_1_0_0`: научный, данные, лицензия, патент, безопасность и приватность. Он нужен для управляемого перехода от `1.0.0-rc.1` к финальному релизу без подмены технической готовности научным доказательством.

## 2. ТЕКУЩИЙ СТАТУС

Release-candidate `1.0.0-rc.1` технически собран: strict-манифест прошел проверку с `ok=true`, `rows=5`, `errors=[]`, `warnings=[]`. Это не финальный релиз и не доказательство антигравитации.

## 4. ПРАВИЛО РЕШЕНИЯ

Финальный `RELEASE_1_0_0` может быть подготовлен только если каждый review-блок имеет решение `ПРОЙДЕНО` или явно зафиксированное решение `ПРОЙДЕНО С ОГРАНИЧЕНИЯМИ`, а все ограничения перенесены в release notes.

Если хотя бы один блок имеет статус `НЕ ЗАВЕРШЕНО`, `БЛОКЕР`, `ОТЛОЖИТЬ` или `ПЕРЕДАТЬ ЮРИСТУ`, финальный release не создается.

## 5. МИНИМАЛЬНЫЙ СОСТАВ ФИНАЛЬНОГО RELEASE

- Финальный manifest без временных значений.
- Strict-отчет проверки.
- Release notes с указанием, что антигравитация не доказана.
- DOI metadata после выбора архива.
- Сводный review-протокол с решениями.
- SHA-256 sums финального пакета.

## 6. ЗАПРЕТЫ

- Не переименовывать `release candidate` в `release`, пока review не закрыт.
- Не утверждать обнаружение эффекта.
- Не публиковать raw/private данные.
- Не раскрывать патентоспособные параметры до решения.
- Не включать секреты, персональные данные и закрытую координацию.

## 7. СЛЕДУЮЩИЕ ДОКУМЕНТЫ

- Финальный `RELEASE_1_0_0` пакет после review.
- DOI/Zenodo metadata с реальным архивным URL после публикации.
- Журнал изменений финального релиза.
- Публичная карточка цитирования и архивная инструкция проверки.
