ДАТА СОЗДАНИЯ: `2026-05-25`.

НАПРАВЛЕНИЕ: `АНТИГРАВИТАЦИЯ`.

СВЯЗАННЫЙ ПАКЕТ: `АНТИГРАВИТАЦИЯ_REVIEW_RELEASE_1_0_0`.

## 1. НАЗНАЧЕНИЕ ДОКУМЕНТА

Этот документ задает порядок закрытия проверки релиза `1.0.0` после создания release-candidate и review-gate пакетов. Он нужен, чтобы финальный выпуск не зависел от устного согласия, впечатления, авторитета, срочности или желания объявить результат.

Антигравитация рассматривается как исследовательское направление, требующее математической формализации, экспериментальной проверки, измеримости и воспроизводимости.

Этот план не утверждает, что антигравитация доказана. Он описывает только процедуру проверки документов, данных, методик, лицензий, патентных рисков, безопасности и приватности.

## 3. ОБЯЗАТЕЛЬНЫЕ БЛОКИ ЗАКРЫТИЯ

Перед финальным решением должны быть закрыты пять блоков:

1. Научная проверка.
2. Проверка данных и воспроизводимости.
3. Лицензионная проверка.
4. Патентная проверка.
5. Проверка безопасности и приватности.

Каждый блок получает один из статусов:

| Статус | Значение |
|---|---|
| `НЕ НАЗНАЧЕНО` | Ответственный не назначен |
| `В РАБОТЕ` | Проверка начата, заключение не готово |
| `ТРЕБУЕТ ИСПРАВЛЕНИЙ` | Есть блокирующие или существенные замечания |
| `ОДОБРЕНО С ОГРАНИЧЕНИЯМИ` | Публикация возможна только с явно описанными ограничениями |
| `ОДОБРЕНО` | Блок не содержит препятствий для выпуска |
| `ОТКЛОНЕНО` | Выпуск в текущем виде запрещен |

## 4. ПОРОГ ДЛЯ ФИНАЛЬНОГО ВЫПУСКА

Финальный `RELEASE_1_0_0` можно готовить только если одновременно выполнены условия:

1. Все пять блоков имеют статус `ОДОБРЕНО` или `ОДОБРЕНО С ОГРАНИЧЕНИЯМИ`.
2. Все блокирующие несоответствия закрыты.
3. Все существенные ограничения перенесены в release notes и публичный disclaimer.
4. SHA-256 опубликованных файлов совпадает с локальными и Bitrix24-копиями.
5. Нет секретов, персональных данных, приватной координации и партнерски чувствительных материалов.
6. Нет текста, который утверждает, что антигравитация доказана.
7. Публичная лицензия применима только к утвержденным публичным материалам.
8. Патентно чувствительные материалы не раскрыты до отдельного решения.

Если хотя бы один пункт не выполнен, результатом должно быть `NO-GO` или `GO С ОГРАНИЧЕНИЯМИ`, но не финальная публикация без ограничений.

## 5. ОПЕРАЦИОННЫЙ ПОРЯДОК

### 5.1. Назначение ответственных

Для каждого блока назначается роль, а не произвольный исполнитель. Пока человек не назначен, в документах используется статус `ТРЕБУЕТ НАЗНАЧЕНИЯ`.

### 5.2. Сбор доказательств

Каждое утверждение о готовности должно иметь ссылку на файл, версию, хэш, протокол, измерение, журнал проверки или explicit negative evidence.

Недопустимо писать `проверено`, если нет записи в журнале доказательств.

### 5.3. Регистрация несоответствий

Любое замечание с риском для научной строгости, данных, лицензий, патентов, приватности или безопасности заносится в журнал несоответствий.

Блокирующее несоответствие запрещает финальный выпуск до закрытия.

### 5.4. Заключения рецензентов

Заключение должно содержать:

- проверенный объем;
- использованные критерии;
- найденные замечания;
- список недопустимых выводов;
- решение по блоку;
- дату;
- идентификатор версии материалов.

### 5.5. Сводное решение

Сводное решение оформляется отдельным протоколом. Решение `GO` не означает доказанность антигравитации. Оно означает только, что конкретный документальный пакет можно выпускать с заявленным уровнем ограничений.

## 6. ЧТО НЕЛЬЗЯ СЧИТАТЬ ЗАКРЫТИЕМ

Нельзя считать review закрытым, если:

- отсутствуют назначенные роли;
- вместо доказательств есть только общая фраза `все хорошо`;
- нет хэшей файлов;
- не проверен публичный и закрытый контур;
- не проверена Bitrix24-копия;
- не описаны отрицательные или нулевые результаты;
- не выделены ограничения;
- гипотеза подана как доказанный физический эффект;
- инженерная компенсация веса названа антигравитацией;
- электромагнитный, тепловой, аэродинамический или вибрационный эффект не отделен от гравитационного вывода.
