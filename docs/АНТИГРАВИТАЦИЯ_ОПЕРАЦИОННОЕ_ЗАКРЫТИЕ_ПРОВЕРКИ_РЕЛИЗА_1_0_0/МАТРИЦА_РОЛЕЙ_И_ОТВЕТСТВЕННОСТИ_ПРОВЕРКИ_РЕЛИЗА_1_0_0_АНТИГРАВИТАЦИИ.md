# МАТРИЦА РОЛЕЙ И ОТВЕТСТВЕННОСТИ ПРОВЕРКИ РЕЛИЗА 1.0.0 АНТИГРАВИТАЦИИ

СТАТУС: `ШАБЛОН НАЗНАЧЕНИЯ РОЛЕЙ`.

ДАТА СОЗДАНИЯ: `2026-05-25`.

ПРОЕКТ: `LANDAU FUND`.

НАПРАВЛЕНИЕ: `АНТИГРАВИТАЦИЯ`.

## 1. НАЗНАЧЕНИЕ

Документ фиксирует роли, которые должны закрыть review-пакет перед финальным выпуском `1.0.0`. Роли не являются утверждением научного результата. Они нужны для контролируемого принятия решения.

Антигравитация рассматривается как исследовательское направление, требующее математической формализации, экспериментальной проверки, измеримости и воспроизводимости.

## 2. ПРИНЦИПЫ НАЗНАЧЕНИЯ

1. Один человек может выполнять несколько ролей только если это явно отмечено как риск независимости.
2. Научная проверка не должна подменяться патентной, лицензионной или организационной проверкой.
3. Роль не закрывается без письменного заключения.
4. Каждое решение должно ссылаться на доказательства.
5. Если ответственный не назначен, статус блока остается `НЕ НАЗНАЧЕНО`.

## 3. ОСНОВНАЯ МАТРИЦА

| Блок | Роль | Ответственный | Обязательные входы | Выход | Текущий статус |
|---|---|---|---|---|---|
| Научная проверка | Научный рецензент | `ТРЕБУЕТ НАЗНАЧЕНИЯ` | Расчетный каркас, лабораторный контур, валидация, репликация, отрицательные результаты | Заключение о научной корректности и ограничениях | `НЕ НАЗНАЧЕНО` |
| Данные и воспроизводимость | Ответственный за данные | `ТРЕБУЕТ НАЗНАЧЕНИЯ` | Манифест, словарь данных, SHA-256, шаблоны измерений, архивные файлы | Заключение о полноте данных и воспроизводимости | `НЕ НАЗНАЧЕНО` |
| Лицензии | Лицензионный проверяющий | `ТРЕБУЕТ НАЗНАЧЕНИЯ` | LICENSE, LICENSE.md, лицензии файлов, атрибуция, публичная граница | Заключение о праве публичной публикации | `НЕ НАЗНАЧЕНО` |

## 4. КРИТЕРИИ НЕЗАВИСИМОСТИ

| Проверка | Минимальный критерий |
|---|---|
| Научная | Рецензент должен иметь возможность написать `NO-GO` без организационного давления |
| Данные | Проверяющий должен перепроверить файлы и хэши, а не только прочитать отчет |
| Лицензии | Проверяющий должен отделить публичные CC BY материалы от private/internal |
| Патенты | Проверяющий должен выделить раскрытия, которые нельзя публиковать до отдельного решения |
| Безопасность | Проверяющий должен считать отсутствие секретов недоказанным, пока не выполнена явная проверка |

## 5. ПРАВА РОЛЕЙ НА БЛОКИРОВКУ

Каждая ключевая роль имеет право поставить `NO-GO`, если найдено блокирующее несоответствие.

Блокирующие причины:

- научно некорректная формулировка;
- утверждение доказанности антигравитации;
- отсутствие воспроизводимых данных при заявлении об экспериментальном результате;
- незакрытый секрет или персональные данные;
- неопределенная лицензия;
- риск преждевременного патентного раскрытия;
- несовпадение хэшей или версий;
- отсутствие публичного disclaimer;
- подмена инженерной компенсации веса антигравитацией.

## 7. ЗАПРЕЩЕННЫЕ ЗАМЕНЫ

Нельзя заменять:

- научное заключение общим одобрением команды;
- проверку данных визуальным просмотром файлов;
- license review наличием файла `LICENSE`;
- patent review общей фразой `патентов нет`;
- security review утверждением `секреты не добавляли`;
- release decision фактом загрузки в GitHub или Bitrix24.

## 8. СЛЕДУЮЩИЙ ДОКУМЕНТ
