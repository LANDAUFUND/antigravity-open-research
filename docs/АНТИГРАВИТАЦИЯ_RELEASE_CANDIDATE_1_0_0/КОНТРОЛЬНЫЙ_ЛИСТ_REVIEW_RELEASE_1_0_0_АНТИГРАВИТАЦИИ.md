# КОНТРОЛЬНЫЙ ЛИСТ REVIEW RELEASE 1.0.0 НАПРАВЛЕНИЯ «АНТИГРАВИТАЦИЯ»

ДАТА: `2026-05-25`.

СТАТУС: `REVIEW GATE`, НЕ ФИНАЛЬНЫЙ РЕЛИЗ.

АНТИГРАВИТАЦИЯ РАССМАТРИВАЕТСЯ КАК ИССЛЕДОВАТЕЛЬСКОЕ НАПРАВЛЕНИЕ, ТРЕБУЮЩЕЕ МАТЕМАТИЧЕСКОЙ ФОРМАЛИЗАЦИИ, ЭКСПЕРИМЕНТАЛЬНОЙ ПРОВЕРКИ, ИЗМЕРИМОСТИ И ВОСПРОИЗВОДИМОСТИ.

## 1. ТЕХНИЧЕСКИЙ REVIEW МАНИФЕСТА

- [x] Манифест содержит обязательные колонки.
- [x] В строках манифеста нет временных полей `ТРЕБУЕТ_*`.
- [x] Каждый путь относителен корню публичного репозитория.
- [x] Для каждого файла рассчитан `РАЗМЕР_БАЙТ`.
- [x] Для каждого файла рассчитан `SHA256`.
- [x] Для каждого файла указан публичный GitHub commit.
- [x] Для каждого файла указан проверенный Bitrix24 file ID.
- [ ] После нового коммита проверить, что публичные GitHub URL открываются на актуальном snapshot.

## 2. НАУЧНЫЙ REVIEW

- [ ] Проверить, что документы не утверждают доказанность антигравитации.
- [ ] Проверить разделение на установленную физику, инженерную компенсацию веса, гипотетические модели и спекулятивные модели.
- [ ] Проверить, что эффект Казимира, ОТО, электромагнитные поля, плазма, сверхпроводники и метаматериалы не подаются как готовое антигравитационное доказательство.
- [ ] Проверить, что UAP/НЛО-контекст не используется как доказательство антигравитации.
- [ ] Проверить, что отрицательные и нулевые результаты считаются валидными научными исходами.

## 3. DATA REVIEW

- [ ] Проверить отсутствие raw/private данных.
- [ ] Проверить, что агрегированные таблицы не позволяют восстановить закрытые параметры стендов.
- [ ] Проверить единицы измерения, погрешности, контрольные серии и правила округления.
- [ ] Проверить, что шум, дрейф, тепловые эффекты, вибрации, электростатика и магнитные наводки не интерпретируются как эффект без контроля.
- [ ] Проверить согласованность CSV-словаря и JSON-схемы публичных агрегатов.

## 4. LICENSE REVIEW

- [ ] Подтвердить `CC BY 4.0` для публичных non-code документов.
- [ ] Подтвердить MIT-лицензию только для файлов кода, где есть явный SPDX.
- [ ] Проверить наличие атрибуции Landau Fund и ссылки на публичный GitHub-репозиторий.
- [ ] Проверить отсутствие сторонних материалов без права публикации.

## 5. PATENT REVIEW

- [ ] Проверить, что публичные документы не раскрывают патентоспособные параметры перспективных стендов.
- [ ] Проверить, что инженерные схемы, режимы, материалы и алгоритмы управления не вынесены в public без решения.
- [ ] Сверить публичный пакет с закрытой патентной картой.
- [ ] Зафиксировать решение: публиковать как open research, отложить, или передать юристу.

## 6. SECURITY И PRIVACY REVIEW

- [ ] Проверить отсутствие паролей, API keys, токенов, cookie, private keys, recovery phrases.
- [ ] Проверить отсутствие персональных данных и закрытой внутренней координации.
- [ ] Проверить, что Bitrix24 file IDs не раскрывают секреты и используются только как технические идентификаторы сверки.
- [ ] Проверить, что закрытый private-lab контур не был случайно опубликован в public.

## 8. ЗАПРЕЩЕННЫЕ ФОРМУЛИРОВКИ

Нельзя писать:

- `антигравитация доказана`;
- `получен антигравитационный двигатель`;
- `эффект подтвержден без независимой репликации`;
- `секретность заменяет публикацию данных`;
- `UAP/НЛО доказывают управляемую метрику`.

Разрешенная формулировка: направление находится в стадии математической формализации, лабораторной проверки, измеримости и воспроизводимости.
