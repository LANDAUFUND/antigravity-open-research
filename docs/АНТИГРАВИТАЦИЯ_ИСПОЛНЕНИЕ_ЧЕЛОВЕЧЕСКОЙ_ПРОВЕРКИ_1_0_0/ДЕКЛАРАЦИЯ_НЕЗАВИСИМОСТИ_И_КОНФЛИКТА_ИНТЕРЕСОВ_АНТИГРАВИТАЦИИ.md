# ДЕКЛАРАЦИЯ НЕЗАВИСИМОСТИ И КОНФЛИКТА ИНТЕРЕСОВ АНТИГРАВИТАЦИИ

СТАТУС: `ШАБЛОН ДЛЯ HUMAN REVIEW`.

ДАТА: `2026-05-25`.

## 1. НАЗНАЧЕНИЕ

Документ задает обязательную декларацию независимости рецензента перед проверкой пакета `RELEASE_1_0_0`.

Антигравитация рассматривается как исследовательское направление, требующее математической формализации, экспериментальной проверки, измеримости и воспроизводимости.

## 2. ДАННЫЕ РЕЦЕНЗЕНТА

```text
РОЛЬ:
ФИО / ИДЕНТИФИКАТОР:
ОРГАНИЗАЦИЯ:
ОБЛАСТЬ КОМПЕТЕНЦИИ:
ДАТА:
```

Персональные данные не должны публиковаться в публичном контуре без отдельного согласия. В публичном GitHub допускается использовать только роль и обезличенный идентификатор.

## 3. ДЕКЛАРАЦИЯ НЕЗАВИСИМОСТИ

Рецензент подтверждает:

- он понимает границу между машинной сверкой и научным review;
- он не будет трактовать наличие файлов, SHA-256 или Bitrix24 ID как доказательство физического эффекта;
- он не будет утверждать, что антигравитация доказана, если не представлены воспроизводимые экспериментальные данные;
- он готов указать ограничения своей компетенции;
- он готов зафиксировать несогласие, неопределенность или `NO-GO`, если основания для `GO` отсутствуют.

## 4. КОНФЛИКТ ИНТЕРЕСОВ

Рецензент должен раскрыть:

- участие в подготовке проверяемых документов;
- финансовую заинтересованность;
- патентную заинтересованность;
- организационную зависимость;
- доступ к закрытым данным, которые могут влиять на решение;
- любые отношения, способные повлиять на независимость вывода.

## 5. РЕШЕНИЕ ПО ДОПУСКУ

| Поле | Значение |
|---|---|
| Конфликт интересов раскрыт | `ДА / НЕТ / НЕ ПРИМЕНИМО` |
| Конфликт блокирует review | `ДА / НЕТ / ТРЕБУЕТ РЕШЕНИЯ` |
| Допуск к review | `ДОПУЩЕН / ДОПУЩЕН С ОГРАНИЧЕНИЯМИ / НЕ ДОПУЩЕН` |
| Кто принял решение | `ТРЕБУЕТ ЗАПОЛНЕНИЯ` |
| Дата решения | `ТРЕБУЕТ ЗАПОЛНЕНИЯ` |

## 6. ЗАПРЕЩЕННЫЕ ВЫВОДЫ

Нельзя писать:

- `конфликт интересов не проверялся, значит его нет`;
- `рецензент назначен, значит его вывод независим`;
- `review можно закрыть без декларации`;
- `релиз можно выпускать без раскрытия патентного и лицензионного конфликта`.
