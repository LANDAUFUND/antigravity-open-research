НАПРАВЛЕНИЕ: `АНТИГРАВИТАЦИЯ`.

## 1. НАЗНАЧЕНИЕ

Пакет переводит машинную сверку и операционное закрытие review `1.0.0` в исполнимую процедуру человеческой проверки: назначение ролей, проверка независимости, передача материалов рецензентам, получение заключений, закрытие `NC-001..NC-006` и заседание комиссии по выпуску.

Антигравитация рассматривается как исследовательское направление, требующее математической формализации, экспериментальной проверки, измеримости и воспроизводимости.

Этот пакет не утверждает, что антигравитация доказана. Он не объявляет финальный `RELEASE_1_0_0` и не заменяет научные заключения специалистов.

## 4. ПРАВИЛО НАЗНАЧЕНИЙ

Конкретные люди не выдумываются и не записываются автоматически. Пока ответственное лицо не назначено реально, поле `СТАТУС` остается `ТРЕБУЕТ_НАЗНАЧЕНИЯ`, а release decision остается `NO-GO ПО УМОЛЧАНИЮ`.

Допустимые роли:

- `НАУЧНЫЙ_РЕЦЕНЗЕНТ`;
- `ОТВЕТСТВЕННЫЙ_ЗА_ДАННЫЕ`;
- `ЛИЦЕНЗИОННЫЙ_ПРОВЕРЯЮЩИЙ`;
- `ПАТЕНТНЫЙ_ПРОВЕРЯЮЩИЙ`;
- `ПРОВЕРЯЮЩИЙ_БЕЗОПАСНОСТИ`;
- `RELEASE_MANAGER`;
- `АРХИВАРИУС`.

## 5. КРИТЕРИИ ПЕРЕХОДА ИЗ `NO-GO`

Переход из `NO-GO ПО УМОЛЧАНИЮ` возможен только если:

1. Все роли назначены.
2. Все назначенные лица подтвердили независимость или раскрыли конфликт интересов.
3. Все материалы переданы и получены.
4. Все заключения получены.
5. Все `NC-001..NC-006` закрыты или явно переведены в допустимые ограничения.
6. Комиссия по релизу приняла одно из решений: `GO`, `GO С ОГРАНИЧЕНИЯМИ`, `NO-GO`, `DEFER`.

## 6. ЗАПРЕЩЕННЫЕ ВЫВОДЫ

Нельзя делать выводы:

- `роли перечислены, значит review закрыт`;

- `назначение release manager равно решению GO`;
- `отсутствие секретов по паттернам равно полной security-проверке`;
- `пакет опубликован в GitHub, значит антигравитация доказана`.
