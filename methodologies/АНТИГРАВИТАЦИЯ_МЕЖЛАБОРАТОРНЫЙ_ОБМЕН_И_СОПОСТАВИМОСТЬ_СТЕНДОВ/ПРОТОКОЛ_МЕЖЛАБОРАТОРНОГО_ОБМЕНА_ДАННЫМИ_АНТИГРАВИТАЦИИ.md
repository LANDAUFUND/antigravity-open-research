# ПРОТОКОЛ МЕЖЛАБОРАТОРНОГО ОБМЕНА ДАННЫМИ АНТИГРАВИТАЦИИ

АНТИГРАВИТАЦИЯ РАССМАТРИВАЕТСЯ КАК ИССЛЕДОВАТЕЛЬСКОЕ НАПРАВЛЕНИЕ, ТРЕБУЮЩЕЕ МАТЕМАТИЧЕСКОЙ ФОРМАЛИЗАЦИИ, ЭКСПЕРИМЕНТАЛЬНОЙ ПРОВЕРКИ, ИЗМЕРИМОСТИ И ВОСПРОИЗВОДИМОСТИ.

## 1. НАЗНАЧЕНИЕ

Протокол задает единый формат передачи данных между лабораториями, чтобы разные группы могли проверять один и тот же результат без потери контекста, единиц, калибровок, условий среды и истории обработки.

## 2. ПРИНЦИП ОБМЕНА

Передается не только итоговый график, а полный пакет:

- паспорт стенда;
- паспорт образца;
- raw data;
- калибровки до и после серии;
- журнал режимов;
- журнал исключений;
- параметры среды;
- версия обработки;
- расчет погрешностей;
- статус рецензии;
- отрицательные и нулевые серии.

## 3. МИНИМАЛЬНЫЙ СОСТАВ ПАКЕТА

| Блок | Обязателен | Комментарий |
| --- | --- | --- |
| `README_СЕРИИ` | да | краткое описание опыта |
| `RAW_DATA` | да | исходные временные ряды |
| `CALIBRATION` | да | калибровки датчиков |
| `ENVIRONMENT` | да | температура, давление, влажность, вибрация |
| `FIELDS` | да | токи, напряжения, `E`, `B`, заземление |
| `PROCESSING` | да | формулы, скрипты или ручной расчет |
| `REVIEW` | желательно | внутренняя или внешняя рецензия |
| `NEGATIVE_RESULTS` | да | отрицательные серии по той же гипотезе |

## 4. ОСНОВНЫЕ ФОРМУЛЫ

```text
P = m · g
ΔP = Pконтроль - Pактивный
Fостаток = ΔP - Fмех - Fэм - Fэлст - Fаэро - Fтепл - Fплазма - Fдатчик
σобщая = √(σслуч² + σсист²)
SNR = |Fостаток| / σобщая
Hdata = SHA256(raw_data)
```

## 5. ПЕРЕМЕННЫЕ

| Переменная | Смысл |
| --- | --- |
| `P` | вес или показание весового канала |
| `m` | масса образца |
| `g` | локальное ускорение свободного падения |
| `ΔP` | разность между контрольным и активным режимом |
| `Fостаток` | остаточная сила после учета известных вкладов |
| `σобщая` | объединенная неопределенность |
| `SNR` | отношение остаточного сигнала к неопределенности |
| `Hdata` | криптографический хэш исходных данных |

## 6. ЕДИНИЦЫ ИЗМЕРЕНИЯ

| Величина | Единица |
| --- | --- |
| масса | кг |
| сила | Н |
| время | с |
| температура | К или °C |
| давление | Па |
| влажность | % |
| вибрация | м/с² |
| магнитная индукция | Тл |
| электрическое поле | В/м |
| напряжение | В |
| ток | А |

## 7. ПРИМЕР ПЕРЕДАЧИ

Лаборатория `LAB-001` передает серию `AG-SERIES-0024`. В пакете есть raw data с хэшем `Hdata`, калибровка весового канала до и после серии, карта магнитного поля, таблица температуры и протокол обработки.

Лаборатория `LAB-002` не пересчитывает график по картинке. Она получает raw data, проверяет хэш, пересчитывает `Fостаток` и сравнивает результат с исходным отчетом.

## 8. ЧТО ПРОВЕРЯЕТ ПРИНИМАЮЩАЯ ЛАБОРАТОРИЯ

- совпадает ли хэш raw data;
- есть ли все обязательные блоки;
- понятны ли единицы измерения;
- воспроизводится ли расчет из переданных данных;
- не исчезают ли отрицательные серии;
- не изменен ли критерий успеха после опыта;
- можно ли повторить опыт по паспорту стенда и образца.

## 9. КАКИЕ ДАННЫЕ НУЖНО СОБРАТЬ

Нужно собрать машинно-читаемые таблицы измерений, файлы калибровки, паспорта приборов, версии программ, логи режимов, фотографии стенда, данные среды, карту полей, журнал исключений, подписи ответственных и хэши всех исходных файлов.

## 10. КАКИЕ ВЫВОДЫ ДЕЛАТЬ НЕЛЬЗЯ

Нельзя делать вывод о независимом подтверждении, если принимающая лаборатория получила только PDF, график, видео, пересказ, обработанную таблицу без raw data или набор файлов без хэшей.

Нельзя объявлять антигравитацию доказанной по факту передачи данных между лабораториями.

## 11. НАУКА ПРОТИВ ПСЕВДОНАУКИ

Научный обмен данными допускает независимый пересчет и обнаружение ошибки. Псевдонаучный обмен передает только удобную витрину результата и блокирует проверку ссылкой на секретность, авторитет или коммерческую тайну.

## 12. СЛЕДУЮЩИЕ ДОКУМЕНТЫ

Нужно подготовить стандарт пакета `AG-EXCHANGE`, таблицу сопоставимости стендов, регламент обмена образцами и публичный шаблон нулевой серии.
